## สิ่งที่ตรงกับ DSS แล้ว
- ใช้ `DB HeaventRounded` เป็น base typography ของระบบ
- ยก base text ขึ้นไปที่ `--text-p` ตาม shell/body ของ DSS
- ใช้ `Sidebar` และ `TopNavbar` ของ DSS runtime จริงใน shell หลัก
- ใช้ spacing/radius/token แบบ semantic แทนการ hardcode ค่าใหม่ใน shell หลัก
- preview ปัจจุบันที่ `http://127.0.0.1:4173/` ตอบ `200`

## สิ่งที่ยังไม่ตรงกับ Storybook 100%
- ยังไม่ได้ import `@uxuissk/design-system/styles.css` เป็น first import ตาม quick start
- ยังไม่ได้ใช้ `AppShellProvider` ตาม inventory ของ MCP
- ยังมี compatibility layer แทนการใช้ runtime component ตรง ๆ หลายตัว
- ยังมีหน้าในระบบอีกจำนวนหนึ่งที่ยังเป็น legacy/custom structure ไม่ได้ migrate ครบทั้งระบบ

## มี issue เพราะอะไร
- **เหตุผลที่ 1: DSS stylesheet integration ยังชนกับ Tailwind v3**
  - MCP และ quick start ระบุว่าต้อง import `@uxuissk/design-system/styles.css` ก่อนทุกอย่าง
  - แต่ใน repo นี้ ถ้าทำตามตรง ๆ จะ build พังเพราะ CSS ของ package ใช้ `@layer base` แล้วชนกับ pipeline ของ Tailwind v3 ในโปรเจกต์
- **เหตุผลที่ 2: MCP/Storybook surface area กว้างกว่าที่ runtime package ใช้ได้จริงในโปรเจกต์**
  - MCP ระบุ component ระดับสูงหลายตัว แต่ runtime package เวอร์ชันที่ใช้อยู่ยังไม่สามารถนำมาใช้ตรง ๆ ในโปรเจกต์นี้ได้ครบ
- **เหตุผลที่ 3: โปรเจกต์ยังต้องคง logic/route/flow เดิมของ BOLA**
  - เลยมีหลายจุดที่ต้อง adapt โครงสร้างเข้าหา DSS แทนการเปลี่ยนเป็น Storybook แบบ 1:1

## ต้องแก้ยังไง
- แก้ upstream หรือวิธี build ให้รองรับ `@uxuissk/design-system/styles.css` กับ Tailwind v3 ได้จริง
- ทำให้ runtime package export และใช้งาน component ระดับสูงตามที่ MCP/Storybook ระบุได้ครบ
- ค่อย ๆ ลด `src/components/ui/ds-compat.tsx` ลงจนเหลือเฉพาะ component ของ DSS runtime จริง
- ทำ page-by-page migration ต่อ โดยเริ่มจากหน้าที่มี traffic สูงก่อน
